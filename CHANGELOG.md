# История изменений

## 1.6.3 — 06.07.2026

### Исправления критического аудита

- Версия приложения повышена до 1.6.3, чтобы интерфейс, release commit и tag снова однозначно идентифицировали исходный код расчёта.
- Фактическое полное закрытие кредита через стратегии “сократить срок” или “снизить платёж” теперь обнуляет будущий платёж по нулевой задолженности, а не по названию стратегии.
- В режиме начисления на конец дня досрочные операции между регулярными платежами доначисляют процент за день операции на остаток после всех операций этой даты.
- Браузерная печать во время отложенного пересчёта больше не показывает устаревший финансовый отчёт: вместо него печатается предупреждение о необходимости дождаться актуального графика.
- Импорт JSON и ссылок строит полный candidate-график до мутации store, поэтому платежи до выдачи, недостаточные total-платежи и чрезмерные generated operations отклоняются транзакционно.
- Расчёт процентов больше не создаёт посуточный массив дат: начисление идёт сегментами по ставкам, льготам и календарным годам, а все даты ограничены календарным горизонтом 120 лет.
- Регулярные правила используют подготовленный платёжный календарь, дедуплицируют месяцы пропуска и ограничивают размер массивов и текстовых полей.

## 1.6.2 — 04.07.2026

### Исправления повторного аудита

- Устранена зависимость порядка досрочных операций в один день от технического ID: порядок теперь хранится отдельной последовательностью и сохраняется при импорте, ссылках и миграции.
- Правило “общий ежемесячный платёж” теперь строится по договорным платёжным датам с учётом первого процентного платежа и продления льготного периода, а не по уже готовым регулярным строкам графика.
- Расчётный хук использует единый отложенный снимок кредита, чтобы график, валюта, выбранный сценарий и настройки округления не расходились между собой во время пересчёта.
- Быстрое включение total-платежа и сохранение total-правил проходят полную проверку до мутации store; причина блокировки показывается прямо в форме платежа.
- Миграция localStorage сортирует историю ставок до обрезки лимита, новые кредиты получают даты относительно текущего дня, а статус localStorage различает успешное сохранение, близость к квоте и ошибку записи.
- Исправлены мелкие проблемы графика: поиск учитывает отложенные проценты и общую задолженность, а строки раскрытого месячного режима сохраняют корректное число колонок.
- Стабилизирован focus trap модальных окон при изменении состояния формы.
- Версия приложения повышена до 1.6.2.

## 1.6.1 — 04.07.2026

- Декомпозирован `App.tsx`: расчёт, импорт, экспорт, ссылки и localStorage/onboarding вынесены в отдельные хуки.
- Исправлено продление длинных льготных периодов: график больше не создаёт финальный balloon payment внутри продолжающейся отсрочки.
- Капитализация процентов теперь применяется для уменьшенного и индивидуального льготного платежа, если платёж не покрыл начисленные проценты.
- В режиме точного изменения ставки операция “снизить платёж” использует ставку, действующую на дату досрочного платежа.
- Добавлен лимит истории изменения ставки: не более 1000 записей в ядре, импорте, ссылках, store и UI.
- Оптимизирован проход по истории ставок при посуточной сегментации процентов.
- Быстрое включение досрочного платежа больше не включает невалидную общую сумму или нулевую операцию без проверки.
- Store отклоняет пересекающиеся льготные периоды до мутации.
- Обзор корректно показывает будущий кредит как ещё не выданный.
- График показывает отложенные проценты и общую задолженность, когда они есть.
- Preview импорта и ссылки форматирует сумму в валюте импортируемого кредита.
- Лимит share-link считается в UTF-8 байтах одинаково при упаковке и распаковке.
- Модальные окна получили dialog-семантику, Escape, focus trap и возврат фокуса.
- Версия приложения повышена до 1.6.1, в метаданные сборки добавлен commit SHA.

## 1.6.0 — 03.07.2026

- Добавлена история изменения процентной ставки с указанием даты начала действия новой ставки.
- Начисление процентов и остаток задолженности теперь рассчитываются с учётом ставки, действующей в соответствующем периоде.
- Аннуитетный платёж пересчитывается после изменения процентной ставки.
- В графике отмечаются строки, в которых начинает действовать новая ставка.
- История ставок сохраняется в локальном хранилище, JSON-экспорте и ссылках на расчёт.
- Добавлен режим применения изменения ставки “точно с даты”: процентный период разрезается на сегменты с разными ставками.
- Расширены регулярные досрочные платежи: неделя, 2 месяца, квартал, полгода, год и общий ежемесячный платёж.
- В льготных периодах теперь настраиваются сумма платежа, начисление процентов и капитализация.
- Печать, импорт и загрузка ссылки показывают историю изменений ставки, а не только базовую ставку.
- Усилена валидация расчётного ядра для повреждённых enum-полей и битых дат.
- Добавлены быстрые кнопки включения и отключения разовых досрочных платежей.
- Цвета компонентов переведены на общую систему переменных для корректного отображения во всех темах.

## 1.5.7 — 02.07.2026

### Исправления аудита и совместимость

- Исправлен пересчёт “снизить платёж” после предыдущих операций “сократить срок”: теперь используется актуальный оставшийся срок, а не исходное количество периодов по договору.
- Разовые досрочные платежи и регулярные правила можно временно отключить отдельным переключателем; такие записи приглушаются в списках, сохраняются, импортируются, но не попадают в расчёт.

- Экспорт для Excel больше не сохраняет HTML-таблицу с расширением `.xls`: файл создаётся как `.html`, чтобы Excel не предупреждал о несоответствии формата и расширения.
- Для двухнедельных и квартальных графиков количество периодов теперь округляется к ближайшему платёжному периоду и не раздувает короткие сроки через `ceil`.
- В `package.json` добавлены GitHub metadata-поля, а `.gitignore` расширен типовыми локальными артефактами окружения.
- Добавлен ESLint с правилами TypeScript и React Hooks, команда `pnpm lint` и запуск lint во всех GitHub Actions workflow.
- Добавлен безопасный адаптер localStorage: если браузер не даёт сохранить данные локально, приложение показывает предупреждение и предлагает JSON-экспорт.
- Добавлены Dependabot, SECURITY.md, CONTRIBUTING.md, favicon, web manifest и CSP meta-тег для статической публикации.
- Добавлен первый компонентный тест React Testing Library для модального окна досрочного платежа.
- README и расчётная документация синхронизированы с фактическими возможностями приложения.

- Добавлены регрессионные тесты для льготного срока, нескольких событий одной датой, audit end-of-day, импорта и лимитов store.

## 1.5.4 — 01.07.2026

### Совместимость старых данных и точность обзора

- Сохранена legacy-семантика старых разовых операций без `amountMode`: если дата совпадает с регулярным платежом, такая запись трактуется как общая сумма строки банка, как и до миграции.
- Остаток долга на главной странице теперь рассчитывается через общий механизм ядра с учётом беспроцентных льготных дней и периодического метода начисления.

- Миграция локального хранилища ограничивает размер старых массивов платежей, правил и льготных периодов до безопасных лимитов.
- JSON-импорт стал строже: нулевые суммы операций и правил, обратные диапазоны правил/льгот и ставка выше 100% отклоняются сразу при загрузке.
- Если кредит закрыт полным досрочным погашением, будущий регулярный платёж сценария показывается как 0.
- Добавлены регрессионные тесты для legacy-операций без `amountMode`, текущего долга, комиссии 0–100%, единовременной комиссии до первого платежа и полного закрытия.

## 1.5.2 — 01.07.2026

- При восстановлении старых данных из локального хранилища нормализуются валюта, даты, срок, типы платежей, округление и настройки начисления процентов.
