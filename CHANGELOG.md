# История изменений

## 1.5.3 — 01.07.2026

### Исправления по повторному ревью

- Версия локального хранилища повышена до 6, чтобы миграция запускалась у пользователей, уже открывавших 1.5.0–1.5.2.
- При восстановлении старых данных теперь нормализуются не только параметры кредита, но и разовые платежи, регулярные платежи и льготные периоды; повреждённые записи с невалидными датами удаляются из расчёта.
- Форма регулярных досрочных платежей больше не позволяет сохранить пустые даты или некорректные месяцы пропуска.
- Режим “общая сумма строки банка” теперь разрешён только в дату регулярного платежа и всегда применяется после регулярного платежа.
- Беспроцентные льготные периоды исключают дни начисления процентов и для досрочных платежей между регулярными датами.
- Добавлена фактическая проверка лимита строк графика перед добавлением каждой строки.
- Уточнены формулировки суммы разового досрочного платежа: комиссия за досрочное погашение удерживается из указанной суммы списания.
- Автоматический release workflow теперь сам собирает архив сайта и создаёт GitHub Release, не полагаясь на запуск второго workflow после push тега.
- Ручной release workflow проверяет формат и существование тега перед сборкой.
- Добавлены регрессионные тесты для миграции localStorage, режима общей суммы и беспроцентной льготы внутри периода.

## 1.5.2 — 01.07.2026

### Устойчивость расчёта и защита от невалидных данных

- Добавлена общая строгая проверка календарных дат для настроек кредита, досрочных платежей, регулярных платежей и льготных периодов.
- Расчётное ядро теперь выполняет fail-fast проверку входных данных и не строит график для нулевого кредита, нулевого срока, отрицательной ставки или невозможных дат.
- Добавлены верхние границы сложности: срок кредита, число досрочных платежей, число сгенерированных регулярных операций, льготные периоды и строки графика.
- Генератор графика теперь выбрасывает ошибку, если кредит не закрывается в допустимое число строк, вместо возврата незавершённого графика.
- Исправлен JSON-экспорт: сохранение JSON больше не запускает построение графика и не ломается из-за ошибок расчёта.
- Импорт JSON запрещает несовместимое сочетание “общая сумма из графика банка” и “сначала досрочный платёж”.
- Исправлена обработка досрочного платежа `earlyFirst` в беспроцентном льготном периоде: отменённые проценты больше не могут быть оплачены досрочной операцией.
- Добавлен Error Boundary, чтобы неожиданные runtime-ошибки не превращались в пустой экран.
- При восстановлении старых данных из локального хранилища нормализуются валюта, даты, срок, типы платежей, округление и настройки начисления процентов.
- Добавлены регрессионные тесты для строгих дат, лимитов, невалидных edge cases и беспроцентного льготного периода.

## 1.5.1 — 01.07.2026

### Ревью, документация и публикация

- Проведён полный технический проход по приложению: TypeScript, тесты, production-сборка и аудит production-зависимостей.
- Основные разделы интерфейса переведены на ленивую загрузку, поэтому основной JS-бандл стал меньше и быстрее отдаётся на статическом хостинге.
- Исправлен конфликт GitHub Actions: автоматический workflow теперь создаёт только тег версии, а архив `dist` и GitHub Release создаёт отдельный release workflow.

- README обновлён под актуальные возможности приложения, GitHub Pages, импорт/экспорт, релизы и проверки.
- Добавлена отдельная документация по архитектуре, расчётной модели и процессу публикации.
