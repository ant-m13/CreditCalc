# Ипотечный калькулятор

Веб-приложение для расчёта ипотечного графика с досрочными погашениями. Работает полностью в браузере: данные хранятся локально на устройстве и не отправляются на сервер.

Проект можно открыть как обычный сайт, в том числе через GitHub Pages.

## Возможности

- расчёт аннуитетных и дифференцированных платежей;
- ежедневное или периодическое начисление процентов;
- настройка базы года: 360, 365, 366, Actual/365 и Actual/Actual;
- первый платёж только процентами, как в банковских графиках;
- досрочные погашения в любую дату, а не только в дату регулярного платежа;
- ввод досрочного платежа как отдельной досрочной части или как полной суммы строки из графика банка;
- стратегии досрочного погашения: сокращение срока, уменьшение платежа, полное закрытие и стратегия из каждой операции;
- автоматический пересчёт будущих платежей и процентов после досрочных погашений;
- редактирование, удаление и сортировка досрочных платежей по дате;
- льготные периоды: отсрочка, только проценты, уменьшенный или индивидуальный платёж;
- комиссии, порог закрытия малого остатка, выбор валюты и округления;
- график платежей в банковском формате: дата, сумма по кредиту, проценты, итого и остаток задолженности;
- первая строка графика — выдача кредита с остатком основного долга;
- итоги по графику за весь срок;
- быстрый переход по графику к дате, месяцу или году, а также раскрытие всего графика;
- остаток долга на текущий день: тело кредита, начисленные проценты и общая сумма;
- сравнение сценариев досрочного погашения и понятный вывод сокращения срока;
- настройка темы, точности денежных сумм и размера шрифта;
- экспорт графика в CSV/XLS, сохранение и загрузка расчёта в JSON;
- печатная версия для сохранения в PDF средствами браузера.

## Локальный запуск

```bash
pnpm install
pnpm dev
```

После запуска Vite покажет локальный адрес приложения в терминале.

Проверки перед публикацией:

```bash
pnpm test
pnpm build
```

## Публикация на GitHub Pages

В проекте уже есть workflow `.github/workflows/deploy-pages.yml`.

Он выполняет:

1. установку зависимостей;
2. запуск тестов;
3. production-сборку;
4. публикацию папки `dist` через GitHub Pages.

Для первого запуска в GitHub откройте `Settings → Pages` и выберите источник `GitHub Actions`. После этого публикация будет выполняться при push в ветку `main` или вручную через вкладку `Actions`.

Адрес опубликованного сайта GitHub покажет после успешного выполнения workflow. Обычно он выглядит так:

```text
https://<owner>.github.io/<repository>/
```

Приложение не требует серверной части, базы данных или переменных окружения.

## Важно

Расчёт является справочной финансовой моделью. Фактический график банка может отличаться из-за правил конкретного договора, операционного дня, округления и внутренних правил банка.
