Браузерное приложение для расчёта кредитного графика, сверки с банковским графиком и планирования досрочных погашений. Приложение работает без backend: расчёты выполняются в браузере, данные сохраняются локально на устройстве.

Проект подходит для публикации как обычный статический сайт, в том числе через GitHub Pages.

## Что умеет приложение

- считать аннуитетные и дифференцированные платежи;
- учитывать ежедневное или периодическое начисление процентов;
- настраивать базу года: 360, 365, 366, фактические дни / 365 дней и фактические дни / фактический год;
- хранить несколько кредитов с отдельными параметрами, платежами, льготными периодами, темой и настройками интерфейса;
- быстро переключаться между кредитами, переименовывать, создавать и удалять кредиты;
- строить банковский график платежей с первой строкой выдачи кредита;
- показывать колонки “по кредиту”, “по процентам”, “итого” и остаток задолженности;
- скрывать колонку “комиссия”, если комиссий в расчёте нет;
- считать итоги по кредиту, процентам, комиссиям и общей сумме за весь срок;
- добавлять разовые досрочные платежи в любую дату;
- указывать разовый досрочный платёж как сумму сверх платежа или как общую сумму списания с комиссией досрочного погашения в дату регулярного платежа;
- временно отключать разовый досрочный платёж отдельным переключателем без удаления записи;
- добавлять регулярные досрочные платежи: раз в неделю, каждый месяц, раз в 2 месяца, квартал, полгода, год, ежегодная премия, процент от регулярного платежа или общий ежемесячный платёж в дату списания;
- задавать дату окончания регулярного платежа и пропускать отдельные месяцы;
- временно отключать регулярное правило отдельным переключателем без удаления записи;
- выбирать стратегию операции: сократить срок, снизить платёж или закрыть кредит полностью;
- сравнивать сценарии: без досрочных, сокращение срока, снижение платежа и “по операциям”;
- пересчитывать будущие платежи и проценты после досрочных погашений;
- раскрывать формулу строки графика: период начисления, число дней, база года, остаток для процентов, проценты до округления, округление и порядок операций;
- учитывать льготные периоды: полная отсрочка, только проценты, уменьшенный или индивидуальный платёж, начисление процентов и капитализацию;
- показывать остаток долга на текущий день: тело кредита, проценты и общую сумму;
- показывать ближайший платёж: дату, итог, тело кредита, проценты, комиссию при наличии и остаток после платежа;
- показывать прогресс погашения и понятное сокращение срока;
- быстро переходить в графике к дате, месяцу или году;
- фильтровать график по году, искать строку по сумме и сворачивать месяцы;
- использовать карточный режим графика на мобильных устройствах;
- настраивать тему, акцентный цвет, ночной режим, масштаб текста и точность отображения сумм;
- экспортировать график в CSV или HTML-таблицу для Excel, сохранять кредит в JSON и загружать JSON обратно;
- предварительно просматривать JSON перед загрузкой и выбирать действие: создать новый кредит или заменить текущий;
- создавать ссылку на расчёт и восстанавливать кредит из такой ссылки;
- проверять обязательные параметры перед созданием JSON, ссылки или кода параметров;
- печатать воспроизводимый отчёт с ключевыми настройками расчёта или сохранять его в PDF средствами браузера;
- показывать версию приложения и страницу “Что изменилось”;
- хранить историю изменения процентной ставки, применять её со следующего платёжного периода или точно с даты изменения и показывать историю ставок в печати, импорте и ссылках.

## Передача расчёта ссылкой

В разделе “Импорт/экспорт” можно создать ссылку на полный расчёт кредита, выбранного в шапке приложения. В ссылку попадают:

- название кредита;
- параметры кредита;
- история изменения ставки;
- разовые и регулярные досрочные платежи;
- льготные периоды;
- выбранный сценарий;
- настройки интерфейса.

График платежей и сравнение сценариев в ссылке не сохраняются. Они пересчитываются локально после открытия.

При открытии ссылки приложение предложит:

1. создать новый кредит из данных ссылки;
2. заменить текущий кредит;
3. отказаться от загрузки.

Важно: любой человек, получивший ссылку, сможет увидеть параметры расчёта. Если операций много и ссылка становится слишком большой, используйте JSON-экспорт. Старые ссылки и JSON-файлы без новых необязательных полей продолжают открываться: приложение подставляет безопасные значения по умолчанию. Если в старом файле или ссылке отсутствует валюта или указано неподдерживаемое значение, она будет заменена на `RUB`, а в окне импорта появится предупреждение; суммы при этом не конвертируются.

## Локальный запуск

Требования:

- Node.js 22;
- pnpm через Corepack.

```bash
corepack enable
pnpm install
pnpm dev
```

После запуска Vite покажет локальный адрес приложения в терминале. В production-режиме расчёт выполняется в Web Worker, чтобы большие графики не блокировали интерфейс; в тестовой среде и старых браузерах используется синхронный fallback.

## Проверки перед публикацией

```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm test:coverage
pnpm build
pnpm audit --prod
```

Что проверяется:

- TypeScript-компиляция;
- статический анализ TypeScript и React Hooks через ESLint;
- тесты расчётного ядра, импорта, экспорта, ссылок и базовых UI-сценариев;
- пороги покрытия Vitest;
- production-сборка;
- известные уязвимости production-зависимостей.

Версия приложения берётся из `package.json`, дата сборки и commit SHA добавляются при production-сборке, а список изменений читается из `CHANGELOG.md`.

Для двухнедельных и квартальных графиков срок в месяцах переводится в ближайшее целое количество платёжных периодов. Например, 1 месяц при двухнедельном графике даёт 2 платежа, а 12 месяцев — 26 платежей.

Если браузер не даёт сохранить данные в `localStorage` (приватный режим, переполненная квота или запрет хранилища), приложение показывает предупреждение, что последние изменения не сохранены. В этом случае можно сразу скачать JSON или повторить сохранение после освобождения места.

При восстановлении старых данных приложение нормализует формат, мигрирует legacy-режим `amountMode: "total"` в `amountMode: "totalWithFee"` и помещает повреждённые кредиты в карантин с отчётом в интерфейсе.

Отключённые разовые платежи и регулярные правила остаются в списках, выделяются приглушённым видом и не попадают в расчёт. Старые записи с нулевой суммой или нулевым процентом также считаются временно отключёнными для совместимости.

## Публикация на GitHub Pages

В проекте есть workflow `.github/workflows/deploy-pages.yml`.

Он выполняет:

1. установку зависимостей;
2. non-blocking audit production-зависимостей;
3. запуск ESLint;
4. запуск тестов;
5. production-сборку;
6. production E2E smoke-test;
7. публикацию папки `dist` через GitHub Pages.

Для первого запуска в GitHub откройте `Settings → Pages` и выберите источник `GitHub Actions`. После этого публикация будет выполняться после успешного auto-release, при push release-тега `v*` или вручную через вкладку `Actions` с указанием существующего тега.

Адрес опубликованного сайта обычно выглядит так:

```text
https://<owner>.github.io/<repository>/
```

Приложение не требует серверной части, базы данных или переменных окружения.

## Релизы с готовым сайтом

Для релизов, публикации и обслуживания используются:

- `.github/workflows/auto-release.yml` — при изменении версии в `package.json` проверяет проект, создаёт git-тег `vX.Y.Z`, собирает архив сайта и создаёт GitHub Release;
- `.github/workflows/deploy-pages.yml` — публикует GitHub Pages только из release-тега после auto-release, прямого tag push или ручного запуска;
- `.github/workflows/release-dist.yml` — ручной или резервный workflow для сборки релиза по уже существующему тегу.
- `.github/dependabot.yml` — еженедельно проверяет обновления npm-зависимостей и GitHub Actions.

Обычный выпуск версии:

```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm test:coverage
pnpm build
git add package.json CHANGELOG.md README.md docs src .github
git commit -m "Release v1.6.9"
git push origin release/1.6.9
```

После merge release-ветки в `main` workflow сам создаст тег, релиз и запустит публикацию GitHub Pages из release-тега. Ручной запуск `release-dist.yml` или `deploy-pages.yml` тоже возможен, но нужно указать существующий тег, например `v1.6.9`.

Архив релиза содержит готовую папку `dist` и подходит для размещения на любом статическом хостинге.

## Ограничения

Расчёт является справочной финансовой моделью. Фактический график банка может отличаться из-за условий договора, операционного дня, правил округления, внутренних банковских правил и особенностей проведения платежей.

Для сверки с банком используйте раскрываемую формулу строки графика и настройки начисления процентов.

Для защиты браузера от ошибочных или чрезмерно больших данных действуют технические лимиты: срок до 1200 месяцев, до 1000 изменений ставки, до 5000 досрочных операций в расчётном ядре, до 5000 правил досрочных платежей, до 100 льготных периодов и до 10000 строк графика.

- React
- TypeScript
- Vite
- Zustand
- Decimal.js
- Recharts
- Vitest

Расчётное ядро находится в `src/loanEngine`, состояние — в `src/store.ts`, нормализация и миграции — в `src/storeNormalization.ts`, запуск синхронного и Worker-расчёта — в `src/loanCalculationRunner.ts`, основные разделы интерфейса — в `src/components`, прикладные сценарии вынесены в `src/hooks`, а `src/App.tsx` связывает маршрутизацию разделов и общую компоновку приложения.
